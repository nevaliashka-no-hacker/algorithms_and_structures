# Проблемы решаемые с помощью деревьев

Главная идея: **дерево помогает хранить данные не просто "в линию", а в виде связей и иерархии**, а также выполнять операции быстрее, чем на обычном списке.

## 1. Хранение иерархии
**Проблема:** данные естественно устроены как "родитель → потомки".

**Примеры:**
- файловая система: папки и файлы
- HTML / DOM в браузере
- структура компании: отдел → команда → сотрудник
- комментарии в соцсетях: комментарий → ответы

**Почему дерево:**
список плохо показывает вложенность, а дерево отражает её напрямую.


## 2. Быстрый поиск, вставка и удаление
**Проблема:** нужно часто искать элементы и поддерживать данные в порядке.

**Решение: BST / AVL / Red-Black Tree**

**Примеры:**
- `TreeMap`, `TreeSet` в языках программирования
- хранение отсортированных пользователей, заказов, цен
- "найди ближайшее число", "все элементы в диапазоне [lo, hi]"

**Почему дерево:**
в сбалансированном дереве поиск, вставка, удаление работают за **$O(log n)$**, а в обычном списке поиск - **$O(n)$**.


## 3. Индексы в базах данных
**Проблема:** данные огромные и лежат на диске, значит важны не только вычисления, но и число обращений к памяти/диску.

**Решение: B-Tree / B+ Tree**

**Где используется:**
- PostgreSQL
- MySQL
- файловые системы

**Что это даёт:**
- быстрый поиск по ключу
- быстрые диапазонные запросы
- мало обращений к диску

**Реальная задача:**
найти все заказы между двумя датами или клиента по ID в базе из миллионов записей.


## 4. Автодополнение и поиск по префиксу
**Проблема:** нужно быстро находить все слова, начинающиеся с заданного префикса.

**Решение: Trie (префиксное дерево)**

**Примеры:**
- подсказки в поиске DuckDuckGo/Google/Yandex
- автодополнение в IDE
- словари и спеллчекеры
- поиск контактов по началу имени

**Почему дерево:**
если хранить слова просто в списке, придётся проверять много строк.
Trie позволяет идти **по символам префикса** и быстро получать подходящие слова.


## 5. Приоритетная обработка задач
**Проблема:** всегда нужно быстро брать "самый важный" элемент.

**Решение: Heap (куча)** - тоже древовидная структура

**Примеры:**
- планировщик задач ОС
- очереди задач на сервере
- алгоритм Дейкстры
- поиск top-k элементов
- обработка событий по времени

**Почему дерево:**
можно быстро получить минимум/максимум: **$O(1)$** на чтение вершины и **$O(log n)$** на вставку/удаление.


## 6. Быстрые запросы на диапазонах
**Проблема:** массив часто меняется, и нужно быстро отвечать на запросы:
- сумма на отрезке
- минимум / максимум
- количество элементов на диапазоне

**Решение: Segment Tree, Fenwick Tree**

**Примеры:**
- аналитика по временным рядам
- игровые рейтинги
- мониторинг метрик
- финансовые данные

**Почему дерево:**
позволяет обновлять значение и отвечать на запросы за **$O(log n)$**, а не пересчитывать всё заново.


## 7. Разбор и выполнение выражений
**Проблема:** нужно понять структуру программы или математического выражения.

**Решение: AST (Abstract Syntax Tree)**

**Примеры:**
- компиляторы
- интерпретаторы
- SQL-парсеры
- калькуляторы выражений

**Почему дерево:**
выражение вроде
`(a + b) * c`
имеет естественную древовидную структуру:
- корень - `*`
- слева - `a + b`
- справа - `c`


## 8. Маршрутизация и поиск по IP-префиксам
**Проблема:** нужно быстро определить, куда отправить пакет по адресу.

**Решение: Prefix Tree / Radix Tree / Patricia Trie**

**Примеры:**
- таблицы маршрутизации в сети
- обработка IP-адресов
- поиск по двоичным префиксам


## 9. Принятие решений
**Проблема:** нужно выбирать действие по набору условий.

**Решение: Decision Tree**

**Примеры:**
- кредитный скоринг
- медицинские рекомендации
- антифрод
- ML-модели


## Простая формула для запоминания

- **Список** - когда данные идут по порядку
- **Хеш-таблица** - когда нужен очень быстрый поиск по точному ключу
- **Дерево** - когда важны **структура, порядок, диапазоны, иерархия или префиксы**
