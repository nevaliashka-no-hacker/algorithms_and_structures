# Семинар 1: "Монотонный стек"

## Блок 1. Разминка — стек и скобки

> Стек — одна из самых простых структур данных. Но задачи, которые он решает, бывают далеко не простыми. Сегодня мы увидим, как один и тот же приём — **монотонный стек** — решает целый класс задач за O(n), где наивные решения требуют O(n²).

### Простейшая реализация стека

В Python коллекция `list` реализована как гетерогенный стек (элементы разного типа).

Собственная реализация стека:

```python
class Stack:
    def __init__(self):
        self.items = []

    def push(self, x):
        """Добавить элемент в конец стека"""
        self.items.append(x)

    def pop(self):
        """Удалить и вернуть последний элемент (выбросит ошибку на пустом стеке)"""
        return self.items.pop()

    def peek(self):
        """Вернуть последний элемент без удаления (выбросит ошибку на пустом стеке)"""
        return self.items[-1]

    def view(self):
        """Вернуть весь стек"""
        return self.items
```


### Задача-разминка: проверка скобочной последовательности

**Условие:** дана строка из символов `(`, `)`, `[`, `]`, `{`, `}`. Определить, является ли она правильной.

```python
def is_valid(s: str) -> bool:
    """
    Идея: открывающую скобку кладём в стек,
    при встрече закрывающей — проверяем, что на вершине
    стека лежит парная ей открывающая.

    Время:  O(n)
    Память: O(n)
    """
    matching = {')': '(', ']': '[', '}': '{'}
    stack = []

    for ch in s:
        if ch in '([{':
            stack.append(ch)
        else:
            # Стек пуст или скобки не совпадают
            if not stack or stack[-1] != matching[ch]:
                return False
            stack.pop()

    return len(stack) == 0


# Демонстрация
tests = ["([]){}", "([)]", "((()))", ")(", ""]
for t in tests:
    print(f"{t!r:>12} -> {is_valid(t)}")
```

**Вывод на экран:**
```
    '([]){}'  -> True
      '([)]'  -> False
    '((()))'  -> True
        ')('  -> False
          ''  -> True
```

## Блок 2. Монотонный стек — идея и Next Greater Element

### 2.1. Что такое монотонный стек

> **Монотонный стек** — это стек, в котором элементы расположены в отсортированном (монотонном) порядке (неубывающем или невозрастающем). При добавлении нового элемента мы **выталкиваем** все элементы, нарушающие монотонность.

Ключевой нюанс: каждый **элемент** добавляется в стек ровно один раз и удаляется из стека не более одного раза → **амортизированная сложность O(n)**.

> **Амортизированная сложность** — это среднее время выполнения одной операции в последовательности, учитывающее как частые быстрые операции, так и редкие медленные. Она гарантирует, что общая стоимость последовательности из `n` операций не превысит `n * амортизированная сложность`, даже если отдельные шаги очень медленные.

```
Пример для невозрастающего стека (decreasing):

Массив: [2, 1, 5, 6, 2, 3]

Шаг 0: push 2          стек: [2]
Шаг 1: push 1  (1<2)   стек: [2, 1]
Шаг 2: pop 1, pop 2,   стек: [5]         ← 5 > 2 и 5 > 1
        push 5
Шаг 3: push 6  (6>5)   стек: [6]         ← pop 5, push 6
Шаг 4: push 2  (2<6)   стек: [6, 2]
Шаг 5: pop 2, push 3   стек: [6, 3]      ← 3 > 2
```

### 2.2. Задача: Next Greater Element

**Условие:** для каждого элемента массива найти **первый элемент правее**, который строго больше текущего. Если такого нет — вернуть `-1`.

```
Вход:  [2, 1, 2, 4, 3]
Выход: [4, 2, 4, -1, -1]
```

#### Наивное решение — O(n²)

```python
def next_greater_naive(nums: list[int]) -> list[int]:
    """Для каждого элемента ищем вправо первый больший — O(n²)."""
    n = len(nums)
    result = [-1] * n
    for i in range(n):
        for j in range(i + 1, n):
            if nums[j] > nums[i]:
                result[i] = nums[j]
                break
    return result
```

#### Решение с монотонным стеком — O(n)

```python
def next_greater(nums: list[int]) -> list[int]:
    """
    Идея: идём справа налево и поддерживаем стек,
    в котором лежат "кандидаты" на ответ — элементы
    в порядке убывания снизу вверх.

    Для текущего nums[i]:
    1) Выкидываем из стека всё, что ≤ nums[i]
       (эти элементы уже никогда не станут ответом
        для элементов левее — nums[i] ближе и больше).
    2) Если стек не пуст — вершина и есть ответ.
    3) Кладём nums[i] в стек.

    Время:  O(n) — каждый элемент push/pop не более 1 раза
    Память: O(n)
    """
    n = len(nums)
    result = [-1] * n
    stack = []  # хранит значения (можно индексы — зависит от задачи)

    for i in range(n - 1, -1, -1):
        # Убираем всех, кто не может быть "next greater"
        while stack and stack[-1] <= nums[i]:
            stack.pop()

        if stack:
            result[i] = stack[-1]

        stack.append(nums[i])

    return result


# Демонстрация
a = [2, 1, 2, 4, 3]
print(f"Массив:       {a}")
print(f"Next greater: {next_greater(a)}")
# Массив:       [2, 1, 2, 4, 3]
# Next greater: [4, 2, 4, -1, -1]
```

#### Визуализация прохода

```
i=4, nums[i]=3  стек: []        → result[4]=-1,   стек: [3]
i=3, nums[i]=4  pop 3           → result[3]=-1,   стек: [4]
i=2, nums[i]=2  2<4, не pop     → result[2]=4,    стек: [4, 2]
i=1, nums[i]=1  1<2, не pop     → result[1]=2,    стек: [4, 2, 1]
i=0, nums[i]=2  pop 1, 2≤2 pop  → result[0]=4,    стек: [4, 2]
```

#### Альтернатива: проход слева направо (хранение индексов)

```python
def next_greater_left_to_right(nums: list[int]) -> list[int]:
    """
    Другой стиль: идём слева направо,
    в стеке храним ИНДЕКСЫ элементов, для которых
    ещё не нашли ответ.

    Когда nums[i] > nums[stack[-1]] — мы нашли ответ
    для элемента на вершине стека.
    """
    n = len(nums)
    result = [-1] * n
    stack = []  # индексы

    for i in range(n):
        while stack and nums[i] > nums[stack[-1]]:
            idx = stack.pop()
            result[idx] = nums[i]
        stack.append(i)

    return result


# Проверка: оба варианта дают одинаковый результат
a = [2, 1, 2, 4, 3]
assert next_greater(a) == next_greater_left_to_right(a)
print("Оба варианта совпадают")
```

> **Важный момент:** два стиля — "справа налево" и "слева направо" — решают одну и ту же задачу. Выбор зависит от вкуса и конкретной задачи. В "слева направо" удобнее хранить индексы.


## Блок 3. Практические задачи

### 3.1. Daily Temperatures

**Условие (LeetCode 739):** дан массив дневных температур. Для каждого дня найти, **через сколько дней** будет более тёплый день. Если не будет — `0`.

```
Вход:  [73, 74, 75, 71, 69, 72, 76, 73]
Выход: [ 1,  1,  4,  2,  1,  1,  0,  0]
```

```python
def daily_temperatures(temperatures: list[int]) -> list[int]:
    """
    По сути — та же задача Next Greater Element,
    но нужен не сам элемент, а РАССТОЯНИЕ до него.

    Поэтому в стеке храним индексы.
    Идём слева направо: если текущая температура выше,
    чем у элемента на вершине стека — нашли ответ,
    расстояние = i - stack.pop().
    """
    n = len(temperatures)
    result = [0] * n
    stack = []  # индексы дней, для которых ищем ответ

    for i in range(n):
        while stack and temperatures[i] > temperatures[stack[-1]]:
            prev_day = stack.pop()
            result[prev_day] = i - prev_day
        stack.append(i)

    return result


temps = [73, 74, 75, 71, 69, 72, 76, 73]
print(f"Температуры: {temps}")
print(f"Ожидание:    {daily_temperatures(temps)}")
# Температуры: [73, 74, 75, 71, 69, 72, 76, 73]
# Ожидание:    [1, 1, 4, 2, 1, 1, 0, 0]
```

### 3.2. Stock Span

**Условие:** дана последовательность цен акций по дням. **Span** для дня `i` — это максимальное количество **подряд идущих дней** (включая текущий), в которые цена была ≤ цены дня `i`.

```
Цены: [100, 80, 60, 70, 60, 75, 85]
Span: [  1,  1,  1,  2,  1,  4,  6]
```

Пояснение для `span[5] = 4`: цена 75, смотрим назад: 60 ≤ 75, 70 ≤ 75, 60 ≤ 75, но 80 > 75 → стоп. Итого 4 дня (включая текущий).

```python
def stock_span(prices: list[int]) -> list[int]:
    """
    Замечание: span[i] = i - j, где j — индекс ближайшего
    дня СЛЕВА с ценой строго больше prices[i].
    Если такого нет, span[i] = i + 1.

    Это задача "Previous Greater Element" — решается
    монотонным стеком (убывающим), проход слева направо.

    В стеке храним индексы; выталкиваем всех, чья цена ≤ текущей.
    """
    n = len(prices)
    span = [0] * n
    stack = []  # индексы; prices[stack[...]] — убывающая последовательность

    for i in range(n):
        # Убираем дни с ценой ≤ текущей
        while stack and prices[stack[-1]] <= prices[i]:
            stack.pop()

        # Если стек пуст — все предыдущие дни имели цену ≤ текущей
        span[i] = (i - stack[-1]) if stack else (i + 1)
        stack.append(i)

    return span


prices = [100, 80, 60, 70, 60, 75, 85]
print(f"Цены: {prices}")
print(f"Span: {stock_span(prices)}")
# Цены: [100, 80, 60, 70, 60, 75, 85]
# Span: [1, 1, 1, 2, 1, 4, 6]
```

### Обобщение паттерна

```
┌─────────────────────────────────────────────────────┐
│  Семейство задач на монотонный стек:                │
│                                                     │
│  • Next Greater Element  (→ правее, строго больше)  │
│  • Next Smaller Element  (→ правее, строго меньше)  │
│  • Previous Greater Element (← левее, строго больше)│
│  • Previous Smaller Element (← левее, строго меньше)│
│                                                     │
│  Отличия только в:                                  │
│    1) направлении обхода (или что храним в стеке)   │
│    2) знаке сравнения при pop (< / ≤ / > / ≥)       │
└─────────────────────────────────────────────────────┘
```

## Блок 4. Сложная задача — Largest Rectangle in Histogram

**Условие (LeetCode 84):** дана гистограмма — массив высот столбцов единичной ширины. Найти площадь **наибольшего прямоугольника**, который можно вписать в гистограмму.

```
Высоты: [2, 1, 5, 6, 2, 3]
Ответ:  10  (прямоугольник высотой 5, шириной 2: столбцы с индексами 2 и 3)
```

```

        █
      █ █
      █ █
      █ █   █
  █   █ █ █ █
  █ █ █ █ █ █
______________
  2 1 5 6 2 3
      ^^^
      10 = 5*2
```

### 4.1. Идея решения

> Для каждого столбца `i` с высотой `h[i]` найдём **максимальный прямоугольник, в котором `h[i]` — самый низкий столбец**. Ширина такого прямоугольника: от **первого столбца слева, который ниже `h[i]`** до **первого столбца справа, который ниже `h[i]`**.

Это означает, что нам нужны:
- `left[i]` — индекс **Previous Smaller Element** (или `-1`)
- `right[i]` — индекс **Next Smaller Element** (или `n`)
- Площадь = `h[i] * (right[i] - left[i] - 1)`

### 4.2. Решение в два прохода

```python
def largest_rectangle_two_pass(heights: list[int]) -> int:
    """
    Шаг 1: для каждого i находим left[i] — индекс ближайшего
            столбца слева, который СТРОГО ниже heights[i].
    Шаг 2: аналогично right[i] — справа.
    Шаг 3: площадь для столбца i = heights[i] * (right[i] - left[i] - 1).

    Время:  O(n)
    Память: O(n)
    """
    n = len(heights)
    left = [-1] * n   # previous smaller
    right = [n] * n    # next smaller
    stack = []

    # Проход слева направо — ищем Previous Smaller
    for i in range(n):
        while stack and heights[stack[-1]] >= heights[i]:
            stack.pop()
        left[i] = stack[-1] if stack else -1
        stack.append(i)

    stack.clear()

    # Проход справа налево — ищем Next Smaller
    for i in range(n - 1, -1, -1):
        while stack and heights[stack[-1]] >= heights[i]:
            stack.pop()
        right[i] = stack[-1] if stack else n
        stack.append(i)

    # Считаем максимальную площадь
    max_area = 0
    for i in range(n):
        width = right[i] - left[i] - 1
        area = heights[i] * width
        max_area = max(max_area, area)

    return max_area


h = [2, 1, 5, 6, 2, 3]
print(f"Высоты: {h}")
print(f"Макс. площадь: {largest_rectangle_two_pass(h)}")
# Макс. площадь: 10
```

### 4.3. Элегантное решение в один проход

```python
def largest_rectangle_one_pass(heights: list[int]) -> int:
    """
    Один проход с "сентинелом" (sentinel).
    Сторожевое значение, обозначающее конец данных.
    https://en.wikipedia.org/wiki/Sentinel_value

    Добавляем 0 в конец массива — это гарантирует,
    что все столбцы будут вытолкнуты из стека.

    Когда столбец выталкивается — мы знаем его правую
    границу (текущий i) и левую границу (новая вершина стека).
    """
    heights = heights + [0]  # сентинел
    stack = [-1]  # сентинел-индекс для левой границы
    max_area = 0

    for i in range(len(heights)):
        while stack[-1] != -1 and heights[i] < heights[stack[-1]]:
            h = heights[stack.pop()]
            w = i - stack[-1] - 1
            max_area = max(max_area, h * w)
        stack.append(i)

    return max_area


# Проверка
assert largest_rectangle_one_pass([2, 1, 5, 6, 2, 3]) == 10
assert largest_rectangle_one_pass([2, 4]) == 4
assert largest_rectangle_one_pass([1]) == 1
print("Все тесты пройдены")
```

## Блок 5. Когда применять монотонный стек

Ключевые "маркеры" в условии задачи:

- "Найти ближайший больший/меньший элемент"
- "Для каждого элемента определить, как далеко он распространяется влево/вправо"
- "Максимальный прямоугольник / максимальная область"
- "Сколько дней/шагов до события"

Сложность:
- Время:  O(n) — каждый элемент push + pop ≤ 1 раз
- Память: O(n) — размер стека

### Шаблон монотонного стека

```python
def monotonic_stack_template(nums: list[int]) -> list[int]:
    """
    Шаблон: Next Greater Element (справа налево).

    Для Next Smaller — поменять > на <.
    Для Previous Greater — идти слева направо.
    Для строгого/нестрогого — менять <= на <.
    """
    n = len(nums)
    result = [-1] * n
    stack = []

    for i in range(n - 1, -1, -1):         # направление
        while stack and stack[-1] <= nums[i]:  # условие монотонности
            stack.pop()
        if stack:
            result[i] = stack[-1]           # или i - stack[-1] для расстояния
        stack.append(nums[i])               # или i для индексов

    return result
```
