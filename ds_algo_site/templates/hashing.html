#️⃣ Хеширование

Что такое хеширование?

Хеширование — это процесс преобразования входных данных произвольного размера в фиксированную строку (хеш) с помощью хеш-функции. Это необратимая операция.

Свойства хорошей хеш-функции:
  • ✅ Детерминированность — одинаковый ввод → одинаковый хеш
  • ✅ Быстрота вычисления
  • ✅ Равномерное распределение — минимизация коллизий
  • ✅ Лавинный эффект — малое изменение ввода сильно меняет хеш

Виды хеширования

Хеш-таблицы (Hash Map)
Ключ → Хеш → Индекс массива. O(1) доступ. Python dict, C++ unordered_map.
Криптографическое хеширование
SHA-256, MD5. Необратимость, стойкость к коллизиям. Безопасность.

Разрешение коллизий

Метод цепочек (Chaining)
Каждая ячейка — связный список. Все коллизии в одну ячейку.
Открытая адресация
Поиск следующей свободной ячейки: линейное, квадратичное, двойное хеширование.

Сравнение реализации на Python, C, C++

Аспект Python C C++
Встроенная хеш-таблица dict — готовая, динамическая Нет встроенной. Реализуется вручную std::unordered_map, std::map
Хеш-функция hash() встроенная Пишется своя (djb2, sdbm и др.) std::hash специализация
Структура таблицы Динамический массив + открытая адресация Массив указателей + связные списки (цепочки) Массив buckets + цепочки (по умолчанию)
Коллизии Открытая адресация (псевдослучайная) Обычно метод цепочек Метод цепочек
Рехэширование Автоматическое при загрузке > 2/3 Ручное, пересоздание массива Автоматическое через rehash()
Крипто-хеши hashlib (SHA-256, MD5) openssl/sha.h библиотека std::hash НЕ криптостойкий, нужны библиотеки

Где используется хеширование в IT?

  • 🗄️ Базы данных: Хеш-индексы для быстрого поиска
  • 🔐 Безопасность: Хранение паролей (bcrypt, Argon2)
  • 🌐 Кеширование: Redis/Memcached — ключ-значение
  • 📦 Контроль целостности: MD5/SHA для проверки файлов
  • 🔗 Блокчейн: Хеш транзакций, майнинг
  • 📋 Git: SHA-1 для идентификации коммитов
  • 🖼️ Дедупликация: Поиск одинаковых файлов
  • 🌍 CDN/Сети: Consistent hashing для распределения
