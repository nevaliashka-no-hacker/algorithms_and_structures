📌 Хеш-функции

Функции, преобразующие данные в числовой индекс для хеш-таблицы.

1. Метод деления (Division Method) ⭐

h(k) = k mod m, где m — размер таблицы (обычно простое число).

СложностьO(1)КачествоХорошее, если m простое

Недостаток: Плохо работает, если m — степень двойки (используются только младшие биты).

2. Метод умножения (Multiplication Method)

h(k) = floor(m * (k * A mod 1)), где A — константа (0 < A < 1), обычно A ≈ (√5-1)/2 (золотое сечение).

СложностьO(1)КачествоНе зависит от выбора m
3. djb2 (алгоритм Дэниела Бернштейна) ⭐

hash = hash * 33 + c — простой и эффективный алгоритм для строк. Широко используется.

СложностьO(n), n — длина строкиКачествоОтличное распределение

Реализация: Начинается с hash=5381, затем для каждого символа hash = ((hash << 5) + hash) + c.

4. sdbm

hash = hash * 65599 + c. Использовался в базе данных sdbm.

СложностьO(n)КачествоХорошее
5. FNV-1a (Fowler-Noll-Vo)

hash = (hash XOR byte) * FNV_prime. Быстрый, хорошее рассеивание. Используется в DNS-серверах.

СложностьO(n)КачествоОтличное
6. Полиномиальное хеширование (Polynomial Rolling Hash) ⭐

hash(s) = s[0]*p^(n-1) + s[1]*p^(n-2) + ... + s[n-1] mod m. Используется в алгоритме Рабина-Карпа, строках.

СложностьO(n)КачествоХорошее при правильном выборе p и m

Важно: Позволяет пересчитывать хеш за O(1) при сдвиге окна — основа строковых алгоритмов.

📌 Методы разрешения коллизий

7. Метод цепочек (Separate Chaining) ⭐

Описание: Каждая ячейка хеш-таблицы содержит связный список (или другое дерево) всех элементов с одинаковым хешем.

ПоискO(1) средний, O(n) худшийВставкаO(1)УдалениеO(1) при двусвязном списке

Плюсы: Простая реализация, неограниченное число элементов. Минусы: Дополнительная память на указатели.

8. Линейное пробирование (Linear Probing) ⭐

Описание: При коллизии ищем следующую свободную ячейку: h(k, i) = (h(k) + i) mod m.

ПоискO(1) среднийВставкаO(1) среднийПроблемаКластеризация

Проблема кластеризации: Образуются длинные непрерывные группы занятых ячеек → замедление.

9. Квадратичное пробирование (Quadratic Probing)

h(k, i) = (h(k) + c1*i + c2*i²) mod m. Уменьшает кластеризацию по сравнению с линейным.

ПоискO(1) среднийПроблемаВторичная кластеризация
10. Двойное хеширование (Double Hashing) ⭐

h(k, i) = (h1(k) + i * h2(k)) mod m. Использует вторую хеш-функцию для шага. Лучший метод открытой адресации.

ПоискO(1) среднийКачествоОтличное распределение

Требование: h2(k) и m должны быть взаимно простыми. Обычно m — степень 2, h2(k) — нечётное.

11. Perfect Hashing (Идеальное хеширование)

Описание: Для статического набора ключей строится хеш-функция без коллизий. Двухуровневая схема: первичная + вторичные таблицы.

ПоискO(1) гарантированныйПрименениеКомпиляторы (ключевые слова)

📌 Операции с хеш-таблицами

12. Вставка (Insert) ⭐

Вычисление хеша → проверка на коллизию → разрешение коллизии → вставка. При превышении load factor — рехэширование.

СложностьO(1) средняя, O(n) худшая с рехэшированием
13. Поиск (Search) ⭐

Вычисление хеша → проверка ячейки → если коллизия, обход по методу разрешения.

СложностьO(1) средняя, O(n) худшая
14. Удаление (Delete) ⭐

Проблема: При открытой адресации простое удаление ломает поиск. Решение: использовать tombstone (маркер "удалено").

СложностьO(1) средняяОсобенностьTombstone в открытой адресации
15. Рехэширование (Rehashing) ⭐

При достижении load factor (обычно 0.75) таблица увеличивается (обычно вдвое) и все элементы вставляются заново.

СложностьO(n)АмортизированнаяO(1) на вставку

📌 Криптографические хеш-функции

16. MD5 (Message Digest 5)

128-битный хеш. Сломан! Не использовать в безопасности, только для проверки целостности несекретных данных.

Размер128 бит (32 hex)СтатусУстарел
17. SHA-1 (Secure Hash Algorithm 1)

160-битный хеш. Сломан! Используется в Git, но для безопасности непригоден.

Размер160 битСтатусУстарел
18. SHA-256 (часть SHA-2) ⭐

256-битный хеш. Текущий стандарт для криптографии, блокчейна (Bitcoin), цифровых подписей.

Размер256 бит (64 hex)СтатусАктуальный
19. SHA-3 (Keccak)

Новый стандарт, отличная от SHA-2 архитектура (sponge construction). Более устойчив к некоторым атакам.

РазмерПеременный (224, 256, 384, 512)СтатусАктуальный
20. Bcrypt / Argon2 (хеширование паролей) ⭐

Не просто хеш! Ключевое отличие: медленные функции с солью (salt) и cost factor для защиты от перебора и rainbow tables.

ОсобенностьНастраиваемая сложностьПрименениеХранение паролей

Всегда используйте bcrypt/Argon2 для паролей, никогда — SHA/MD5!

📌 Продвинутые техники хеширования

21. Consistent Hashing ⭐

Описание: Распределение ключей по кольцу хешей. При добавлении/удалении сервера перемещается минимум ключей. Используется в CDN, распределённых БД.

ПрименениеРаспределённые системы, кешированиеПримерAmazon Dynamo, Cassandra
22. Bloom Filter (Фильтр Блума) ⭐

Описание: Вероятностная структура для проверки "возможно, есть" или "точно нет". Использует несколько хеш-функций и битовый массив.

ВремяO(k), k — число хеш-функцийПамятьO(m) бит

Применение: Проверка паролей (не скомпрометирован ли), кеш, БД.

23. Cuckoo Hashing

Две хеш-таблицы с разными функциями. Если место занято — вытесняем элемент в другую таблицу. Гарантирует O(1) worst-case для поиска.

24. Алгоритм Рабина-Карпа (поиск подстроки) ⭐

Использует rolling hash для поиска подстроки: хеш окна пересчитывается за O(1), затем сравнивается с хешем образца.

СложностьO(n + m) средняя, O(n*m) худшаяПамятьO(1)
25. Merkle Tree (Дерево Меркла)

Каждый лист — хеш данных, каждый внутренний узел — хеш дочерних хешей. Корень (root hash) подтверждает целостность всех данных.

ПрименениеБлокчейн (Bitcoin, Ethereum), Git, BitTorrent
