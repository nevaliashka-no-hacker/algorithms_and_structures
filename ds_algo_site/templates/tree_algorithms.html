📌 Обходы дерева (Tree Traversals)

1. In-order обход (Левый → Корень → Правый) ⭐

Описание: Рекурсивный обход: сначала левое поддерево, затем корень, затем правое. Даёт отсортированный порядок в BST.

Сложность времениO(n)Сложность памятиO(h) стек вызовов, h — высота

Применение: Получение отсортированного массива из BST, валидация BST.

Языки: Python — рекурсия / yield from | C — void inorder(Node*) | C++ — рекурсия или стек

2. Pre-order обход (Корень → Левый → Правый) ⭐

Описание: Сначала корень, затем левое, затем правое поддерево. Используется для копирования дерева.

Сложность времениO(n)Сложность памятиO(h)

Применение: Сериализация дерева, префиксная запись выражений, клонирование.

Языки: Python — список + рекурсия | C — printf в начале | C++ — лямбда-функция

3. Post-order обход (Левый → Правый → Корень) ⭐

Описание: Сначала оба поддерева, потом корень. Используется для удаления дерева (сначала дети, потом родитель).

Сложность времениO(n)Сложность памятиO(h)

Применение: Удаление дерева, вычисление размера папки, постфиксная запись выражений.

4. Level-order обход / BFS (По уровням) ⭐

Описание: Обход по уровням с использованием очереди (FIFO).

Сложность времениO(n)Сложность памятиO(w), w — максимальная ширина уровня

Применение: Поиск кратчайшего пути в невзвешенном графе, сериализация, печать по уровням.

Языки: Python — collections.deque | C — очередь на связном списке | C++ — std::queue

📌 Поиск в деревьях

5. Поиск элемента в BST ⭐

Описание: Использует свойство BST (левый < корень < правый). На каждом шаге отсекается половина дерева.

Сложность времениO(h), в среднем O(log n), худший O(n)Сложность памятиO(1) итеративно, O(h) рекурсивно

Совет для собеседования: Покажите и итеративную, и рекурсивную версию.

6. Поиск минимума / максимума в BST

Описание: Минимум — самый левый узел, максимум — самый правый.

Сложность времениO(h)Сложность памятиO(1)
7. Поиск LCA — наименьшего общего предка ⭐

Описание: Поиск самого глубокого узла, который является предком двух заданных узлов.

Сложность времениO(n) для обычного дерева, O(h) для BSTСложность памятиO(h)

Часто на собеседованиях! Для BST — используйте свойство порядка, для обычного — рекурсивный поиск.

📌 Модификация дерева

8. Вставка в BST ⭐

Описание: Поиск места + добавление листа. Не нарушает свойства BST.

Сложность времениO(h)Сложность памятиO(1) итеративно, O(h) рекурсивно
9. Удаление узла из BST ⭐⭐ (самый сложный базовый)

Описание: Три случая: 1) лист — просто удалить, 2) один ребёнок — заменить на ребёнка, 3) два ребёнка — найти преемника (мин в правом) и заменить.

Сложность времениO(h)Сложность памятиO(h)

Обязательно знать на собеседовании! Случай с двумя детьми — самый важный.

10. Зеркальное отражение (Invert Binary Tree) ⭐

Описание: Поменять местами левое и правое поддерево для каждого узла. Знаменитый вопрос (тот самый из Twitter).

Сложность времениO(n)Сложность памятиO(h)

📌 Свойства дерева

11. Вычисление высоты (глубины) дерева

Описание: Рекурсивно: 1 + max(height(left), height(right)).

Сложность времениO(n)Сложность памятиO(h)
12. Проверка сбалансированности

Описание: Проверить, что для каждого узла |height(left) - height(right)| ≤ 1.

Сложность времениO(n)Сложность памятиO(h)

Совет: Возвращайте -1 как сигнал несбалансированности, чтобы избежать двойного обхода.

13. Подсчёт количества узлов

Описание: 1 + count(left) + count(right), для полного дерева можно через формулу 2^h - 1.

Сложность времениO(n) обычное, O(log n * log n) для полногоСложность памятиO(h)
14. Проверка, является ли дерево BST

Описание: Проверить, что каждый узел лежит в допустимом диапазоне (min, max).

Сложность времениO(n)Сложность памятиO(h)

Хитрость: Используйте in-order обход — значения должны строго возрастать.

📌 Построение деревьев

15. Построение BST из отсортированного массива

Описание: Рекурсивно берём середину как корень, левую половину — левое поддерево, правую — правое. Строит сбалансированное BST.

Сложность времениO(n)Сложность памятиO(log n)
16. Построение дерева из Pre-order и In-order обходов

Описание: Pre-order даёт корень, In-order показывает, какие узлы слева и справа.

Сложность времениO(n)Сложность памятиO(n) для хеш-таблицы индексов
17. Построение выражения из постфиксной записи

Описание: Используется стек: операнды в стек, при встрече оператора создаём узел с двумя операндами.

Сложность времениO(n)Сложность памятиO(n)

📌 Балансировка

18. Повороты AVL-дерева (LL, RR, LR, RL)

Описание: Четыре типа вращений для восстановления баланса после вставки/удаления.

Сложность времениO(1) на один поворотСложность памятиO(1)

Типы: LL — правый поворот, RR — левый поворот, LR — левый+правый, RL — правый+левый.

19. Балансировка Красно-чёрного дерева (перекраска + повороты)

Описание: После вставки/удаления применяются правила: красный узел не может иметь красного родителя, все пути имеют одинаковое число чёрных узлов.

Сложность времениO(log n) амортизированноеСложность памятиO(1)

📌 Продвинутые алгоритмы

20. Сериализация и десериализация бинарного дерева ⭐

Описание: Преобразование дерева в строку и обратно. Обычно используют pre-order с маркерами для null.

Сложность времениO(n)Сложность памятиO(n)

Часто на собеседованиях! Используйте любой обход, но с маркерами для пустых узлов.

21. Morris Traversal (обход без памяти) ⭐

Описание: In-order обход с O(1) дополнительной памяти путём временного изменения указателей дерева (threaded binary tree).

Сложность времениO(n)Сложность памятиO(1) — уникально!

Козырь на собеседовании! Покажите, что знаете способ обхода без рекурсии и без стека.

22. Диаметр дерева

Описание: Длина самого длинного пути между двумя узлами (не обязательно через корень).

Сложность времениO(n)Сложность памятиO(h)

Формула: diameter = max(left_height + right_height, left_diameter, right_diameter)

23. Максимальная сумма пути

Описание: Найти путь с максимальной суммой значений узлов. Путь может начинаться и заканчиваться в любом узле.

Сложность времениO(n)Сложность памятиO(h)

📌 Алгоритмы для специальных деревьев

24. Вставка в Trie (префиксное дерево)

Описание: Постепенное создание узлов для каждого символа строки. Последний узел помечается как конец слова.

Сложность времениO(k), k — длина словаСложность памятиO(k) на слово
25. Поиск в Trie (автодополнение)

Описание: Поиск префикса, затем сбор всех слов из поддерева. Основа поисковых подсказок.

Сложность времениO(p + m), p — длина префикса, m — число совпаденийСложность памятиO(1) + вывод
26. Heapify (построение кучи из массива)

Описание: Преобразование массива в бинарную кучу (min/max) за O(n) просеиванием снизу вверх.

Сложность времениO(n)Сложность памятиO(1) in-place

Важно: Не O(n log n), а именно O(n) — доказывается математически!

27. Код Хаффмана (построение дерева)

Описание: Построение оптимального префиксного дерева для сжатия данных на основе частот символов с использованием min-heap.

Сложность времениO(n log n)Сложность памятиO(n)
