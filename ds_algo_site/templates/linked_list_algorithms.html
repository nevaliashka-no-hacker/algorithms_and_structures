⚙️ Алгоритмы связных списков

🎯 Связные списки — вторая по популярности тема на собеседованиях после массивов! Обязательно знайте Two Pointers.

📌 Базовые операции

1. Вставка в начало (Prepend) ⭐

Создать узел, его next → текущий head, head → новый узел.

ВремяO(1)ПамятьO(1)
2. Вставка в конец (Append) ⭐

С tail указателем: O(1). Без tail: пройти весь список O(n).

ВремяO(1) с tail / O(n) безПамятьO(1)
3. Удаление по значению ⭐

Найти узел, изменить указатель предыдущего на следующий, удалить узел.

ВремяO(n)ПамятьO(1)
4. Поиск элемента ⭐

Линейный проход от head до нахождения значения или NULL.

ВремяO(n)ПамятьO(1)

📌 Техника двух указателей (Two Pointers) ⭐⭐⭐

Самая важная техника для связных списков на собеседованиях!

5. Поиск середины списка (Floyd's Tortoise and Hare) ⭐⭐⭐

Медленный указатель (1 шаг) и быстрый (2 шага). Когда быстрый в конце → медленный на середине.

ВремяO(n)ПамятьO(1)
6. Обнаружение цикла (Cycle Detection) ⭐⭐⭐

Те же быстрый и медленный указатели. Если встретились → цикл есть. Один из топ-вопросов!

ВремяO(n)ПамятьO(1)
7. Поиск начала цикла ⭐⭐

После обнаружения цикла: один указатель с головы, другой с места встречи. Двигаются по 1 шагу → встреча в начале цикла.

ВремяO(n)ПамятьO(1)
8. N-й узел с конца ⭐⭐

Два указателя: первый на n шагов вперёд, затем оба двигаются. Когда первый в конце → второй на N-м с конца.

ВремяO(n)ПамятьO(1)

📌 Разворот и модификация

9. Разворот списка (Reverse Linked List) ⭐⭐⭐

Итеративно: три указателя (prev, curr, next). Меняем направление ссылок. Обязательный алгоритм!

ВремяO(n)ПамятьO(1) итеративно / O(n) рекурсивно
10. Разворот подсписка (Reverse Sublist)

Разворот между позициями m и n. Требует сохранения узла перед началом разворота.

ВремяO(n)ПамятьO(1)
11. Удаление N-го узла с конца ⭐⭐

Найти N-й с конца (алгоритм 8) + изменить указатели для удаления.

ВремяO(n)ПамятьO(1)
12. Удаление дубликатов из отсортированного списка

Проход с одним указателем: если текущий == следующий → пропустить следующий.

ВремяO(n)ПамятьO(1)

📌 Слияние и объединение

13. Слияние двух отсортированных списков (Merge) ⭐⭐⭐

Два указателя, берём меньший узел, двигаем указатель. Часть Merge Sort.

ВремяO(n + m)ПамятьO(1)
14. Пересечение двух списков (Intersection) ⭐⭐

Выровнять длины (пройти более длинный на разницу), затем идти синхронно до совпадения узлов.

ВремяO(n + m)ПамятьO(1)
15. Чередование (Zipper / Alternate merge)

Поочерёдно брать узлы из двух списков. Используется в перетасовке.

ВремяO(n + m)ПамятьO(1)

📌 Продвинутые алгоритмы

16. Палиндром (Palindrome Linked List) ⭐⭐

Найти середину (алг. 5) → развернуть вторую половину (алг. 9) → сравнить две половины.

ВремяO(n)ПамятьO(1)
17. Сортировка связного списка (Merge Sort) ⭐⭐

Разделить пополам (алг. 5) → рекурсивно сортировать → слить (алг. 13). Лучший выбор для списков!

ВремяO(n log n)ПамятьO(log n) стек рекурсии
18. Перестановка в группы по K (Reverse in K-Groups) ⭐⭐⭐

Разворот группами по K узлов. Сложный алгоритм, часто на собеседованиях.

ВремяO(n)ПамятьO(1)
19. Копирование списка со случайным указателем (Deep Copy) ⭐⭐

Каждый узел имеет next и random. Решение: вставить копии между оригинальными узлами.

ВремяO(n)ПамятьO(1) без хеш-таблицы
20. LRU Cache (Least Recently Used) ⭐⭐⭐

Двусвязный список + хеш-таблица. O(1) get и put. Очень часто на собеседованиях!

ВремяO(1) обе операцииПамятьO(capacity)
