⚙️ Алгоритмы очередей и стека

🎯 Стек и очередь — основа для многих алгоритмов (BFS, DFS, парсинг, окна). Знайте паттерны!
📑 Содержание:
Паттерны стека (7)
Паттерны очереди (5)
Deque / Sliding Window (4)
Priority Queue (4)

📌 Паттерны и алгоритмы стека

1. Проверка скобок (Valid Parentheses) ⭐⭐⭐

Каждую открывающую → в стек, закрывающую → сверяем с вершиной. В конце стек должен быть пуст.

ВремяO(n)ПамятьO(n)

Самый частый вопрос на собеседованиях!

2. Вычисление выражений (Evaluate Expression) ⭐⭐

Парсинг инфиксной/постфиксной записи. Стек операндов + стек операторов (или один стек для RPN).

3. Next Greater Element (Следующий больший) ⭐⭐⭐

Monotonic Stack (убывающий): проходим массив, пока элемент > вершины стека → извлекаем (это ответ для них).

ВремяO(n)ПамятьO(n)
4. Min Stack (Стек с минимумом) ⭐⭐

Поддерживать дополнительный стек минимумов. При push — минимум из нового и текущего минимума. O(1) на getMin().

5. Реализация стека через две очереди

При push — во вторую очередь, затем перелить первую во вторую, поменять их местами. Pop — из первой очереди.

6. Удаление смежных дубликатов

Идём по строке: если символ == вершина стека → pop, иначе push. Удаляет пары одинаковых символов.

7. Largest Rectangle in Histogram ⭐⭐⭐

Monotonic Stack (возрастающий): находим для каждого столбца границы, где он минимален.

ВремяO(n)ПамятьO(n)

📌 Паттерны очереди

8. Реализация очереди через два стека ⭐⭐

Два стека: входной и выходной. Enqueue → push во входной. Dequeue → если выходной пуст, перелить входной в выходной (реверс), затем pop.

EnqueueO(1)DequeueАмортизированное O(1)
9. Реализация циклической очереди (Circular Queue) ⭐⭐

Массив фиксированного размера, два указателя front и rear. Использовать modulo для зацикливания.

10. BFS с очередью ⭐⭐⭐

Обход в ширину: очередь хранит узлы текущего уровня. Ключевой алгоритм для графов и деревьев.

ВремяO(V + E)ПамятьO(V)
11. Очередь с использованием связного списка

Head для dequeue, tail для enqueue. Все операции O(1).

12. Перемешивание очереди (Interleave Queue)

Разделить очередь пополам, чередовать элементы: первый из первой половины, первый из второй, второй из первой...

📌 Deque и Sliding Window

13. Sliding Window Maximum ⭐⭐⭐

Deque хранит индексы элементов окна в порядке убывания значений. При движении окна: удалить вышедшие, добавить новый.

ВремяO(n)ПамятьO(k), k — размер окна
14. Sliding Window Minimum

Аналогично, но Deque в порядке возрастания.

15. First Negative in Window

Храним в Deque только отрицательные числа окна, извлекаем первое для каждого положения.

16. Palindrome Check с Deque

Добавить все символы в Deque, затем сравнивать popLeft и popRight.

📌 Priority Queue (Heap)

17. K-й наибольший элемент (Kth Largest) ⭐⭐⭐

Min-heap размера K. Проходим массив → если размер > K, удаляем минимум. В конце вершина кучи — K-й наибольший.

ВремяO(n log k)ПамятьO(k)
18. Top K частых элементов ⭐⭐

Подсчитать частоты (HashMap) → min-heap по частоте размера K.

ВремяO(n log k)ПамятьO(n)
19. Merge K Sorted Lists ⭐⭐⭐

Min-heap хранит головы всех списков. Извлекаем минимум, добавляем его следующий элемент в кучу.

ВремяO(N log k), N — всего элементов, k — число списковПамятьO(k)
20. Median Finder (Find Median from Data Stream) ⭐⭐⭐

Две кучи: max-heap для левой половины, min-heap для правой. Медиана — вершина одной из куч.

ВставкаO(log n)Поиск медианыO(1)
