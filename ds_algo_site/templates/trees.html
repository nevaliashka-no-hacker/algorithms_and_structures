🌳 Деревья в структурах данных

Что такое дерево?

Дерево — это иерархическая нелинейная структура данных, состоящая из узлов (nodes), соединенных ребрами (edges). В отличие от линейных структур (массивы, списки), деревья отражают иерархические отношения.

Ключевые термины:
  • Корень (Root) — верхний узел дерева
  • Родитель (Parent) — узел, имеющий потомков
  • Потомок (Child) — узел, на который ссылается родитель
  • Лист (Leaf) — узел без потомков
  • Поддерево (Subtree) — любой узел со своими потомками
  • Глубина (Depth) — длина пути от корня до узла
  • Высота (Height) — максимальная глубина дерева

Виды деревьев

Бинарное дерево
Каждый узел имеет ≤ 2 потомков. Основа для BST, куч, выражений.
BST (двоичное дерево поиска)
Левый потомок < родитель < правый потомок. Быстрый поиск O(log n).
AVL-дерево
Самобалансирующееся BST, разница высот ≤ 1.
Красно-чёрное дерево
BST с цветовой балансировкой. Меньше вращений, чем AVL.
B-дерево
Сбалансированное дерево для дисковых хранилищ и баз данных.
Trie (префиксное)
Для хранения строк. Автодополнение, проверка орфографии.

МетодПорядок обходаПрименение

Post-orderЛевый → Правый → КореньУдаление дерева

Где используются деревья в IT?

  • 🗄️ Базы данных: B-деревья и B+-деревья в индексах MySQL, PostgreSQL
  • 📁 Файловые системы: NTFS, ext4 используют деревья для организации
  • 🌐 Парсинг: DOM-дерево в браузерах, абстрактное синтаксическое дерево
  • 🤖 ИИ/ML: Деревья решений, Random Forest, XGBoost
  • 🗜️ Сжатие: Код Хаффмана
  • 🔍 Автодополнение: Trie-структуры в поисковиках
  • 🎮 Игры: Деревья поведения для AI, дерево сцен для рендеринга
